## Challenge Overview

The Apify $1M Challenge is designed to reward developers who build and publish high-quality automation tools, web scrapers, MCP servers, and AI agent tools on the Apify Store. Developers can earn both immediate challenge prizes and ongoing passive income from users who run their Actors.

**Key Opportunity:** AI is doing more work than ever, but the tools are still missing. This is your chance to build automation workflows, web data extractors, MCP servers, and new tools for AI agents to fill this gap.

**What is Apify Store?**
- Marketplace of 8,000+ AI and automation tools called "Actors"
- 25,000 customers actively using the platform
- In September alone, Apify paid out $563,000 to community developers
- Developers earn regular passive income from customers who use their Actors

## Eligibility & Rules

### Who Can Participate
- **Individuals or teams** are welcome to participate
- Must register using the same email as your Apify account (or sign up later)
- If your email links to both personal and organization accounts, the **organization is registered by default**

### Actor Eligibility
- Only the **first 5 Actors** you publish after registration are eligible for the challenge
- Actors must be published **after November 3, 2025** to be eligible
- Actors published before November 3, 2025 are **NOT eligible** for challenge rewards
- Each eligible Actor can earn up to $2,000 from the prize pool
- All Actors must meet the minimum technical criteria (see Technical Requirements section)

### Registration Requirements
- Sign up using the official registration form
- Agree to Challenge Terms including restrictions on scrapers targeting listed services
- Use the same email address as your Apify account

## Technical Requirements

All eligible Actors must meet these minimum criteria:

### 1. Documentation
- **Unique, comprehensive README** that clearly explains:
  - What the Actor does
  - How to use it
  - Input parameters
  - Output format
  - Use cases

### 2. Input Schema
- Must have a **defined input schema** OR operate in **standby mode**
- Input schema helps validate user inputs and improve reliability

### 3. Output Schema
- Must have a **defined output schema**
- Provides predictable, structured data output (typically JSON)

### 4. Actor Quality Score
- **Minimum score of 65/100** required
- Score is evaluated across seven categories:
  - **Reliability**: Run success rate, automated testing, error handling
  - **Popularity**: User engagement, number of runs, saves, repeat usage
  - **Feedback & Community**: User reviews, ratings, responsiveness to issues
  - **Ease of Use**: Clear titles, descriptions, intuitive interface
  - **Pricing Transparency**: Predictable, clear pricing models
  - **History of Success**: Developer track record
  - **Congruency**: Consistency across components (titles, descriptions, schemas)
- Score updates multiple times daily
- View your score in Apify Console under Insights > Actor Quality

### 5. Overall Quality Standards
- Must pass Apify's hygiene and quality evaluation
- No restrictive third-party licenses
- Properly handles errors and edge cases
- Follows Apify platform best practices

## Prize Structure

### Grand Prize (Top 3 Participants - Chosen by Jury)
- **First Place:** $30,000
- **Second Place:** $20,000
- **Third Place:** $10,000

Winners selected after the challenge finishes based on cumulative performance.

### The Big $1M Prize Pool (User-Based Rewards)
- **$2 per monthly active user** for each eligible Actor
- **Minimum payout:** $100 (at 50 users)
- **Maximum payout:** $2,000 (at 1,000 users per Actor)
- Up to 5 Actors per participant = **up to $10,000 total** from this pool
- **Not guaranteed:** Maximum pool of $920,000 distributed until reached

### Weekly Spotlight Prize
- **$2,000 awarded each week**
- Selected by:
  - Industry experts
  - Community vote
  - Jury decision
- Criteria: Brilliant code, amazing concept, or especially great user experience
- Multiple opportunities throughout the challenge period

### Post-Challenge Revenue
- Continue earning passive income after the challenge ends
- Add pay-per-event monetization
- Join the open-source fair share program

## Judging Criteria

### Weekly Spotlight Awards
Evaluated on one or more of:
- **Code Excellence:** Clean, well-structured, maintainable code
- **Innovative Concept:** Creative solution to real problems
- **User Experience:** Intuitive interface, clear documentation, ease of use
- Selected by industry experts, community vote, or jury

### Grand Prize Evaluation
Winners chosen based on:
1. **Cumulative Monthly Active Users:** Total engagement across all eligible Actors
2. **Actor Quality Scores:** Performance across all seven quality categories
3. **Technical Excellence:** Code quality, architecture, innovation
4. **Problem-Solving Value:** How well the Actor addresses real user needs
5. **Overall Impact:** Contribution to the Apify ecosystem

### What "Monthly Active User" Means
- A user who ran your Actor in the 30 days before calculation
- Excludes internal/test accounts
- Counts unique users, not number of runs

## Prohibited Activities & Restrictions

### Prohibited Actor Types
The following Actors are **NOT eligible** for rewards:

1. **Restrictive License Actors:**
   - Actors using third-party software with restrictive licenses

2. **Specific Scraper Restrictions:**
   - YouTube scrapers
   - LinkedIn scrapers
   - Instagram scrapers
   - Facebook scrapers
   - TikTok scrapers
   - X (Twitter) scrapers
   - Apollo.io scrapers
   - Amazon scrapers
   - Google Maps scrapers
   - Google Search scrapers
   - Google Trends scrapers
   - **Note:** Non-scraping functions for these platforms ARE eligible

3. **Business Model Restrictions:**
   - Rental Actors
   - Pay-per-result (PPR) Actors

### Prohibited Activities
Participants will be disqualified for:
- **Fraud or metrics gaming:** Artificial inflation of user counts or engagement
- **Spamming:** Excessive promotion in Apify community channels
- **Code reuse:** Reusing old code or creating duplicates specifically for the challenge
- **Plagiarism:** Copying existing Actors without significant improvement

### Fair Use Guidelines
- **Can you use AI to code an Actor?** Yes, but ensure the final Actor meets all quality criteria
- **Copying existing Actors:** Avoid unless you can **significantly improve** them
- You must validate demand via Google Trends, competitor analysis, and user interviews
